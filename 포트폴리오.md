# Industry Insight Agent 포트폴리오 버전

## 1. 프로젝트 한 줄 소개

비정형 산업 뉴스 데이터를 자동 수집하고, LLM으로 분류·요약하여, 분석 가능한 구조화 데이터와 대시보드로 연결한 업무 자동화 시스템입니다.

## 2. 이 프로젝트를 어떻게 포지셔닝할 것인가

이 프로젝트는 단순 뉴스 분석 프로젝트가 아니라 아래 흐름을 구현한 자동화 시스템으로 설명하는 것이 가장 좋습니다.

`비정형 데이터 입력 -> 자동 수집/정제 -> LLM 처리 -> 구조화 저장 -> 분석/시각화 -> 의사결정 지원`

즉, 핵심은 "뉴스를 잘 분류했다"가 아니라 아래 3가지입니다.

- 비정형 데이터를 실무에서 쓸 수 있는 구조화 데이터로 변환했다
- LLM을 단발성 호출이 아니라 파이프라인 안에 넣었다
- 사람이 하던 반복 업무를 자동화 가능한 시스템으로 만들었다

## 3. 햄프킹과의 연결 포인트

햄프킹에서 강조될 가능성이 높은 키워드와 본 프로젝트의 연결은 아래처럼 정리할 수 있습니다.

### AI Agent

- 입력 데이터를 받아 단계별로 처리하는 파이프라인을 설계했다
- 수집, 분류, 요약, 저장, 분석이라는 역할이 분리된 구조다
- 향후 알림, 액션 실행, RPA 연동까지 확장 가능한 형태로 만들었다

### RPA 자동화

- 사람이 매일 뉴스를 읽고 분류하고 요약하는 반복 업무를 자동화 대상으로 정의했다
- 자동 수집 계층과 결과 저장 구조를 넣어 후속 자동화에 바로 연결할 수 있게 했다
- 결과 CSV는 다른 시스템 입력값이나 보고 자동화의 중간 산출물로 활용할 수 있다

### 비정형 데이터 처리

- 뉴스 텍스트를 정규화하고 분류/요약해 구조화 데이터로 전환했다
- 단순 정형 데이터 분석이 아니라 텍스트 중심 데이터 처리 역량을 보여준다
- 분류 체계와 요약 결과를 통해 검색·분석·모니터링이 가능한 형태로 바꿨다

### 업무 프로세스 구조화

- 문제를 데이터 수집, 전처리, AI 처리, 저장, 분석, 시각화 단계로 나눠 설계했다
- 운영 관점에서 입력과 출력이 명확한 파이프라인을 만들었다
- fallback 경로를 둬서 API가 없거나 네트워크가 제한돼도 데모가 가능하다

## 4. 포트폴리오에서 강조해야 할 핵심 메시지

### 강조 1. "LLM 활용"보다 "LLM 내재화"

이 프로젝트는 단순히 프롬프트를 한 번 던진 것이 아니라, LLM을 실제 처리 단계 안에 내재화한 시스템이라는 점을 강조하는 것이 좋습니다.

### 강조 2. "분석 프로젝트"보다 "자동화 프로젝트"

이 프로젝트는 pandas 분석이 중심이 아니라, 사람이 하던 뉴스 모니터링 업무를 자동화 가능한 형태로 바꾼 것이 핵심입니다.

### 강조 3. "텍스트 처리"보다 "실무 활용 가능한 구조화"

뉴스 텍스트를 category, summary, source, processed_at 같은 필드로 저장해 후속 분석과 리포팅에 쓰일 수 있도록 만든 점을 강조하세요.

### 강조 4. "기능 구현"보다 "업무 설계 능력"

문제를 기능 단위가 아니라 업무 흐름 단위로 나눠 설계했다는 점이 중요합니다.

## 5. 시스템 설명 문구

면접이나 포트폴리오 소개에서 아래 문구를 그대로 써도 자연스럽습니다.

> 산업 뉴스 모니터링 업무를 자동화하기 위해, 뉴스 수집부터 LLM 분류·요약, 결과 저장, 트렌드 분석, 대시보드 시각화까지 이어지는 end-to-end 파이프라인을 구현했습니다.

짧게 말할 때는 이렇게 줄일 수 있습니다.

> 비정형 뉴스 데이터를 구조화해 의사결정에 연결하는 LLM 기반 자동화 시스템입니다.

## 6. 문제 정의를 더 실무적으로 말하는 방법

기존 방식:

- 분석가가 여러 소스의 뉴스를 직접 확인
- 제목과 본문을 읽고 주제를 수작업으로 분류
- 핵심 내용을 요약해 별도 문서나 시트에 정리

문제점:

- 데이터 양이 많아 누락 가능성이 높다
- 분류 기준이 사람마다 달라 일관성이 떨어진다
- 반복 작업이 많아 고부가가치 분석에 시간을 쓰기 어렵다

개선 방향:

- 수집 자동화
- LLM 기반 분류/요약 자동화
- 분석 가능한 구조로 저장
- 대시보드 기반 모니터링

## 7. 데모 시나리오

발표나 과제 제출에서는 아래 순서로 보여주면 좋습니다.

1. 자동 수집 또는 저장된 CSV를 입력으로 선택한다
2. 파이프라인 실행으로 뉴스가 정규화되고 분류/요약되는 과정을 보여준다
3. 생성된 `result.csv`에서 구조화된 필드를 보여준다
4. Streamlit에서 카테고리 분포, 산업별 비교, 키워드 분석을 보여준다
5. 사람이 하던 반복 업무가 어떻게 시스템화됐는지 설명한다

## 8. 코드 기준으로 실제 강조 가능한 구현 포인트

- `src/data/collect_news.py`
  자동 수집 계층. live RSS 우선, 실패 시 local seed fallback 구조
- `src/pipeline.py`
  수집부터 분석까지 연결하는 메인 오케스트레이션
- `src/llm/classify_news.py`
  뉴스 카테고리 분류 엔진
- `src/llm/summarize_news.py`
  핵심 내용 요약 엔진
- `app.py`
  자동화 결과를 보여주는 운영형 대시보드

## 9. 햄프킹 맞춤 어필 문장

### 버전 A

비정형 텍스트 데이터를 단순히 분석하는 수준을 넘어, 실제 업무 프로세스 안에서 자동으로 수집·처리·저장·시각화되는 구조로 만들었다는 점이 이 프로젝트의 핵심입니다.

### 버전 B

LLM을 활용해 사람이 하던 뉴스 분류와 요약 업무를 자동화하고, 후속 데이터 활용이 가능하도록 구조화된 결과를 생성하는 파이프라인을 설계했습니다.

### 버전 C

AI 모델 호출 자체보다 중요한 것은 이를 업무 자동화 흐름 안에 배치하는 것이라고 생각했고, 이 프로젝트에서 그 구조를 end-to-end로 구현했습니다.

## 10. 면접에서 물어볼 가능성이 높은 질문과 답변 포인트

### 왜 이 프로젝트를 했나요

- 비정형 데이터 처리와 업무 자동화를 같이 보여줄 수 있는 주제였기 때문
- 단순 분석이 아니라 실무형 시스템 설계 역량을 보여주고 싶었기 때문

### 단순 뉴스 요약기와 무엇이 다른가요

- 수집, 정제, 분류, 요약, 저장, 분석, UI까지 이어지는 전체 파이프라인이라는 점
- 결과를 구조화해 후속 분석과 자동화에 연결할 수 있다는 점

### 햄프킹 업무와 어떤 관련이 있나요

- 비정형 데이터 처리
- 반복 업무 자동화
- AI 기능을 실제 시스템으로 연결하는 설계 경험

## 11. 앞으로 더 확장하면 좋은 방향

- 실시간 뉴스 스트리밍
- 감성 분석 추가
- 중요 이벤트 알림 자동화
- RPA 또는 사내 툴 입력 연동
- 주간 리포트 자동 생성

## 12. 최종 요약

이 프로젝트는 "뉴스 분석"이 아니라 "비정형 데이터 기반 업무 자동화 시스템"으로 설명할 때 가장 강합니다.

햄프킹 관점에서 보면 특히 아래 역량을 보여줍니다.

- LLM 활용 능력
- 자동화 파이프라인 설계 능력
- 비정형 데이터 처리 능력
- 분석 결과를 실제 사용 가능한 형태로 연결하는 능력
